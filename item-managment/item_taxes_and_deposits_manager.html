Enable Deposits and Taxes Automation

To activate this feature, go to Tools >> Settings >> Sales Order In the Automation section, enable the "Calculate and Apply Item Taxes or Deposits to Sales Order" option

  • Customers must have the state populated in their address, and the format should be in two uppercase letters, such as CA, CT, MA, NY.

Bottle Deposits

1. Create deposit item 

To record the calculated deposit amount in the sales order, you need to create Non-Catalog item for each deposit type. Follow these steps:

  • Create a non-inventory item in ERP/QB or a new item in the LodDepot account.
  • Assign the item as a Non-Catalog item.
  • This item will be used to collect selected deposits for the specified state.

2. Bottle Item Setup

*Important
 When item base unit is Case or Box provide  Item Base Unit Count (quantity of bottles in case) 

  • The deposit quantity will be calculated by multiplying the Order Quantity by the Items in the Base Unit and applied to orders where the customer's state matches the deposit state.
  • When the quantity of bottle items is modified, added, or removed, the system will automatically recalculate the deposit amount.

3. Add Deposit to Items Taxes and Deposit Manager

The final step is to create a deposit type inside the Taxes and Deposit Manager 

  1. Navigate to Item Management → Item Taxes and Deposits → click Add
  2. Name your Deposit and select Tax/Deposit Type (1) → Bottle Deposit in this case
  3. Enter Deposit Amount per bottle (1)
  4. Select the Tax/Deposit Item (2)
  5. This deposit is applied on State basis, therefore write in the appropriate State (3)
    *State must be written as an abbreviation and in capital letters
  6. Make sure that you make this deposit Active (4). Since we are applying to a bottle deposit only to certain items, do not select Apply to Entire Order

7. After you create the deposit it will appear in Item Taxes and Deposits Manager. Last step is to add items, on which the bottle deposit should be applied. Select the bottle deposit and click Add to List (5) on the right hand side of the screen. Make sure to select all applicable items. 

Item Taxes

1. Create deposit item 

To record the calculated tax amount in the sales order, you need to create or assign a Non-Catalog item for each tax type. Follow these steps:

  • Create a non-inventory item in ERP/QB or a new item in the LodDepot account.
  • Assign the item as a Non-Catalog item (1)
  • This item will be used to collect taxes.

2. Create a Tax Type in Items Taxes and Deposits Manager

The final step is to create a tax type inside the Taxes and Deposit Manager 

  1. Navigate to Item Management → Item Taxes and Deposits → click Add
  2. Name your Tax and select Tax/Deposit Type (1) → Sugar Tax in this case
  3. Enter Tax Amount per base unit amount  (1)
  4. Select the Tax/Deposit Item (2)
  5. This deposit is applied on State basis, therefore write in the appropriate State (3)
    *State must be written as an abbreviation and in capital letters
  6. Make sure that you make this deposit Active (4). Since we are applying to a bottle deposit only to certain items, do not select Apply to Entire Order

7. After you create the tax it will appear in Item Taxes and Deposits Manager. Last step is to add items, on which the tax should be applied. Select the tax and click Add to List on the right hand side of the screen. Make sure to select all applicable items. 

Delivery Fee

1. Create a delivery fee item 

To record the calculated tax amount in the sales order, you need to create or assign a Non-Catalog item for each tax type. Follow these steps:

  • Create a non-inventory item in ERP/QB or a new item in the LodDepot account.
  • Assign the item as a Non-Catalog item (1)
  • This item will be used to collect delivery fees for that specific State, Route or Region.

2. Create a Delivery Fee Type in Items Taxes and Deposits Manager

The final step is to create a Delivery Fee type inside the Taxes and Deposit Manager 

  1. Navigate to Item Management → Item Taxes and Deposits → click Add
  2. Name your Delivery Fee and select Tax/Deposit Type → Delivery Fee in this case
  3. Enter Fee Amount per entire order
  4. Select the Tax/Deposit Item → Delivery Fee that we created in Step 1
  5. This fee can be applied based on several criteria: State, Tax Category, Route, Region
    *State must be written as an abbreviation and in capital letters
  6. Make sure that you make this deposit Active (4). Since we are applying the Delivery Fee only to the entire order, select Apply to Entire Order

Creating Tax Category, Route, Region

State, Tax Category, Route and Region all act as criteria based on which the tax/deposit or a fee are applied to Sales Orders of a Customer. Unlike States, you can create more Tax Categories, Routes and Regions. 

All three can be Criteria can be created in Settings
Tools → Settings → Tables 

Select a Table Name and click Add

Applying Criteria to Customers 

Navigate to Customers/Vendors → Customers, select a customer and click Edit

Tax Categories (1) (more than can be applied)
Regions (2)

Routes (3)

How to Remove Tax/Deposit or a Fee from a Sales Order?

You can remove fees, taxes or deposits from existing orders. Navigate to Sales Orders → select the order → click Edit 

In the comments section write the following: 

NOFEE - will remove the applied delivery fee
NOTAX - will remove the applied tax
NODEPOSIT - will remove bottle deposit
* To remove all fees, taxes and deposits, you have to write all three codes in the comments.
→ Save the order and the code will be applied. To reverse, remove the code from the comments.
